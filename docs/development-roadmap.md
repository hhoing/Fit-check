# Development Roadmap

## 1. 현재 MVP 안정화

- 현재 구현된 텍스트/URL 입력 흐름을 정리합니다.
- 공고 카드, 상세 모달, 상태 필터가 안정적으로 동작하는지 확인합니다.
- 데이터 모델을 앞으로 확장하기 쉽게 정리합니다.

## 2. 데이터 모델 확장

- 원본 URL 저장
- 개인 메모 저장
- 우선순위 저장
- 지원일 저장
- 알림 예정일 저장
- 입력 출처 저장: text, url, image

## 3. 이미지 입력 추가

- 입력 모드에 이미지 업로드를 추가합니다.
- 업로드한 이미지에서 텍스트를 추출합니다.
- 추출된 텍스트를 기존 공고 파싱 API 흐름과 연결합니다.

## 4. 사용성 개선

- 마감 임박순 정렬
- 검색
- 상태별 필터 개선
- 즐겨찾기 또는 우선순위 표시
- 메모 입력 및 수정

## 5. 알림 기능

- 앱 안에서 마감 임박 공고를 강조합니다.
- 브라우저 알림을 검토합니다.
- 캘린더 연동은 이후 단계로 둡니다.

## 6. 저장 방식 개선

- localStorage에서 DB 저장으로 이전할지 검토합니다.
- Supabase, SQLite, Postgres 중 프로젝트 규모에 맞는 방식을 선택합니다.
- 로그인 기반 사용자별 저장은 DB 이전 이후 검토합니다.

## 7. 확장 기능

- 현재 보고 있는 공고를 저장하는 브라우저 확장 프로그램
- 북마클릿
- 회사 후기 링크 개선
- 통근 시간 실제 API 연동
